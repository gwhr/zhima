# billing-system: 需求规格

## 套餐配置

| 字段 | 说明 |
|------|------|
| 价格 | 套餐售价（元） |
| Opus 额度 | 可用于 Opus 4.6 调用的金额（元） |
| 修改次数 | 代码/论文修改次数上限 |
| 预览次数 | 在线预览次数上限 |
| 有效期 | 套餐生效天数 |

## 套餐明细

| 套餐 | 价格 | Opus 额度 | 修改次数 | 预览次数 | 有效期 |
|------|------|-----------|----------|----------|--------|
| 基础版 | 49 元 | 25 元 | 10 次 | 3 次 | 30 天 |
| 标准版 | 99 元 | 50 元 | 50 次 | 不限 | 90 天 |
| 旗舰版 | 199 元 | 100 元 | 不限 | 不限 | 永久 |

## 额度分配逻辑
- 用户购买套餐后自动创建 UserQuota 记录
- UserQuota 关联 userId、workspaceId、套餐类型、额度值、剩余额度、过期时间

## API 接口

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | /api/user/quota | 查看当前额度和阶段（normal/tightened/economy） |
| GET | /api/user/usage | 消耗明细列表，支持分页 |
