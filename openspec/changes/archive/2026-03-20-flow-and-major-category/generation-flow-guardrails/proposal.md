# 生成流程闸门与防误操作

## 目标
修复“论文生成结束前点击预览报错”问题，并把生成链路收敛为“代码生成 -> 预览确认 -> 论文生成”。

## 背景
- 当前在论文生成中提前点击预览，会触发依赖缺失导致页面报错。
- 代码生成期间操作按钮可重复点击，容易造成重复触发和状态混乱。
- 已确认需求后仍可“重新调整并评估”，与“进入生成阶段后锁定需求”的预期不一致。
- 论文生成缺少“预览通过”前置条件，流程上存在跳步。

## 范围
- 前端工作空间详情页增加流程闸门与按钮状态控制。
- 后端接口增加强校验，防止绕过前端限制。
- 补齐预览相关依赖，修复构建时报错。
- 新增协作规则：功能改动需先走 PR。
