# Proposal: Workspace Flow Alignment, Progress Feedback, and Runtime Preview

## Why

User feedback showed three major friction points:

1. topic recommendations and selected tech stack were inconsistent
2. generation waiting state lacked clear progression and actionability
3. users needed a controllable way to "run and see effect" before download

In addition, long AI chat responses needed better editing controls and login needed lower friction on trusted devices.

## Scope

1. Align topic recommendation prompts with selected major/tech stack.
2. Improve workspace generation progress messaging and waiting state feedback.
3. Improve generated scaffold packaging/readability for low-experience users.
4. Add AI chat stop/cancel action.
5. Introduce runtime preview queue/session mechanism with time-limited slots.
6. Add trusted-device login retention to reduce repeated SMS usage.

## Out of scope

1. Full per-project isolated runtime sandbox for arbitrary stacks (future phase).
2. Payment/commercial strategy changes (handled by separate archives).
