# 功能确认闸门（Feature Confirmation Gate）

## 目标

在工作空间详情页的“生成代码/生成论文”前增加一层“功能确认”流程：

1. 用户先确认当前功能范围，或提交“修改想法”让 AI 重新分析需求。
2. 功能确认后，先执行难度评估，再进入代码生成流程。

## 背景

当前流程中，用户创建完工作空间后可直接点击“生成代码”，但在真实使用中常出现以下问题：

- 需求范围未最终确认，生成后需要反复返工。
- 用户临时补充需求（新增角色、模块、表数量约束）缺少结构化入口。
- 难度评估与生成流程脱节，用户无法在生成前得到复杂度反馈。

## 范围

- 工作空间详情页新增“功能确认”卡片与“需求修改”对话框。
- 新增后端 API：支持“确认当前需求”与“按用户补充意见重分析需求”。
- 在确认阶段输出并落库难度评估结果（分项评分+建议）。
- 代码生成按钮增加前置校验：未确认需求时不可直接生成。
