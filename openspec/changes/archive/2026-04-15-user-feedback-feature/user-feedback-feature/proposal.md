# Proposal: User Feedback Feature Backfill

## Why

The codebase already includes a user feedback channel, but the corresponding OpenSpec archive was missing.  
To keep project handover and future changes understandable, OpenSpec needs to reflect the current delivered behavior.

## Scope

1. Record the existing user-side feedback submission and history flow.
2. Record the existing admin-side feedback list, search, pagination, and status update flow.
3. Record image upload limits, protected image access, and audit-log expectations.

## Out of scope

1. New notification strategy after feedback is resolved.
2. New feedback taxonomy, tags, or SLA workflow.
3. Any product redesign beyond documenting the current shipped behavior.
