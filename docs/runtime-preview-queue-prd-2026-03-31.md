# PRD：运行预览排队与限时会话（已落地）

日期：2026-03-31  
状态：已实现（本地已验证）

## 1. 背景
- 用户需要“看得到效果”的在线运行预览，而不是只看静态代码文件。
- 平台资源有限，必须做并发控制和会话回收，避免单个用户长期占用。

## 2. 本次目标
- 提供“运行预览”入口，支持排队。
- 每次会话限时，倒计时结束自动回收。
- 未充值用户仅可免费运行 1 次；充值后不受该限制，可继续排队。
- 队列、并发、免费次数在前端可见。

## 3. 规则（已实现）
- 单用户免费运行次数：默认 1 次（环境变量可配）。
- 是否已充值判断：存在 `TokenLedger.type=RECHARGE` 且 `deltaAvailablePoints > 0`。
- 免费次数计算口径：用户历史 `TaskJob(type=PREVIEW)` 的 `PENDING/RUNNING/COMPLETED` 总数。
- 并发槽位：全局并发上限控制，超过后进入队列。
- 会话时长：默认 5 分钟，结束后自动标记完成并释放槽位。
- 队列顺序：按任务创建时间 FIFO。

## 4. 后端实现
- 新增服务文件：
  - `lib/runtime-preview.ts`
- 新增 API：
  - `GET /api/workspace/:id/runtime-preview`：查询运行预览状态
  - `POST /api/workspace/:id/runtime-preview`：提交运行预览任务
- 复用已有 `TaskJob`（`type=PREVIEW`）承载运行预览队列与状态。
- `preview-build` API 新增 `runtime=1&jobId=...` 模式：
  - `PENDING` 返回排队页（自动刷新）
  - `RUNNING` 返回可交互预览页 + 顶部剩余时长倒计时
  - 非运行态返回会话结束提示页

## 5. 前端实现
- 组件：
  - `components/code-preview-dialog.tsx`
- 增加“运行预览”页签：
  - 展示并发上限、排队人数、单次时长、免费次数
  - 支持“启动运行预览”“刷新状态”
  - 展示当前任务状态：排队中 / 运行中 / 已结束
  - 运行中显示剩余倒计时
  - 受限时显示阻塞原因（如“未充值仅一次已用完”）

## 6. 环境变量
- `RUNTIME_PREVIEW_SECONDS`（默认 300，最小 60）
- `RUNTIME_PREVIEW_MAX_CONCURRENCY`（默认 1）
- `RUNTIME_PREVIEW_FREE_LIMIT`（默认 1）

## 7. 验收标准
- 未充值账号：
  - 首次可启动运行预览
  - 首次结束后再次启动被阻止，并提示充值后可排队
- 运行中：
  - 前端显示“运行中”与剩余时间
- 队列中：
  - 前端显示队列位置和队列总数
- 会话结束：
  - 自动回收并显示“上次运行会话已结束”

## 8. 后续增强建议
- 真容器沙箱（Node/SpringBoot）替换当前预览页面模式。
- 增加管理员运行预览监控（活跃会话、排队长度、超时率）。
- 增加运行预览扣点规则（按分钟或按次）。
