# 选题推荐“再来一批”功能 PRD（2026-03-28）
## 1. 背景
- 当前“选题推荐”默认只返回一批题目，用户经常遇到“方向对但题目不满意”的情况。
- 现有流程缺少“同关键词继续扩展题目”的入口，用户只能返回重搜，体验割裂。

## 2. 目标
- 在不更换关键词的前提下，支持继续获取更多题目。
- 尽量避免重复题目，提升推荐多样性和可选空间。

## 3. 范围（P0）
- 用户端创建工作空间弹窗（`pick-topic` 步骤）新增“再来一批”按钮。
- 选题推荐接口支持补充推荐参数：
  - `batch`（第几批）
  - `excludeTitles`（已推荐题目，避免重复）
- 前端把新批次题目追加展示，并保持已选逻辑。

## 4. 交互规则
- 首次搜索：展示第 1 批推荐。
- 点击“再来一批”：请求下一批并追加展示。
- 若没有新题目：提示“没有更多推荐，可换关键词再试”。
- 按钮在请求中禁用，避免重复点击并发请求。

## 5. 验收标准
- 同一关键词下可连续拉取多批题目。
- 第二批及后续批次与前批次重复率明显降低（通过 `excludeTitles` 控制）。
- 请求失败时有清晰错误提示，不影响用户手动输入题目继续流程。

## 6. 非目标
- 本次不做“按行业标签筛选”或“按难度滑杆筛选”。
- 本次不做推荐结果持久化历史。
