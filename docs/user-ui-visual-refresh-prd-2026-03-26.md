# PRD：用户端视觉升级（2026-03-26）

## 1. 背景与问题

当前用户端功能完整，但视觉层存在以下问题：

1. 页面层级不清晰，第一眼重点不突出。
2. 导航 active 态弱，信息密度高时可读性下降。
3. 认证页与业务页风格割裂，品牌感不足。
4. 演示录屏时“质感”不足，影响整体可信度。

## 2. 目标

1. 在不改后端逻辑的前提下，统一用户端视觉语言。
2. 提升关键页面（登录、仪表盘、工作空间）的信息层级与可读性。
3. 保证现有业务流程和交互行为完全兼容。

## 3. 非目标

1. 不调整 API 协议、数据库结构、任务调度逻辑。
2. 不改管理员端功能流程。
3. 不在本期引入复杂主题配置系统（仅做视觉基线升级）。

## 4. 目标用户

1. 首次访问并准备创建项目的新用户。
2. 高频使用工作空间推进“代码+论文”链路的用户。
3. 需要录屏演示系统能力的用户（答辩/路演场景）。

## 5. 功能范围

### P0（本次已实现）

1. 全局主题 token 升级（颜色、字体、背景氛围）。
2. 用户端壳层升级（导航条 + 侧栏 + 主体容器）。
3. 仪表盘首页重构（Hero、统计卡、快速入口）。
4. 工作空间列表页重构（头部、空态、卡片反馈）。
5. 工作空间详情页容器级视觉统一（不改生成逻辑）。
6. 登录/注册页面风格统一。

### P1（下一迭代建议）

1. 工作空间详情页操作步骤区改为更清晰的时间线视觉。
2. 用户端品牌主题可选（2~3 套预设，仅 token 切换）。
3. 落地页与用户控制台之间增加更连贯的过渡引导。

## 6. 交互与体验要求

1. 关键入口（新建项目、登录/注册）在首屏可见。
2. 导航 active 态在桌面端明显可识别。
3. 卡片 hover 与按钮反馈需有轻量动效，避免“静态感”。
4. 空状态必须可操作，不能只提示“暂无数据”。

## 7. 验收标准（DoD）

1. `/login`、`/register`、`/dashboard`、`/workspace`、`/workspace/:id` 页面视觉统一。
2. 现有登录、注册、工作空间创建/删除、任务生成功能可正常使用。
3. TypeScript 检查通过：`npx tsc --noEmit`。
4. 不引入新的后端接口或数据库迁移。

## 8. 风险与缓解

1. 风险：样式变更可能影响局部布局适配。
   缓解：优先保持 DOM 结构不变，只做 class 层改造。
2. 风险：Windows 构建环境存在历史性 `next build` worker 异常。
   缓解：本次以 `tsc` + 页面手测为准，后续在 CI/Linux 环境复核 build。
