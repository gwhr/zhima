# 工作空间代码生成流程优化 PRD（2026-03-27）

## 1. 背景
- 当前工作台流程中，用户在未生成代码前也能看到 AI 对话，容易产生“无上下文修改”的误操作。
- 代码生成完成后仍可重复点击“生成代码”，可能造成重复任务与体验混乱。
- 代码预览面板中缺少按结构分组（后端/前端/SQL）能力，用户难以判断工程完整性。
- 对于“用户端 + 管理端”双端毕设场景，生成约束不够明确，导致偶发只产出单端示例。

## 2. 目标
- 将工作台主链路收敛为“功能确认 -> 生成代码 -> 预览确认 -> 生成论文”。
- 将 AI 对话定位为“代码产出后的讨论与增量修改工具”。
- 提升代码产物可理解性与下载认知一致性。
- 强化双端场景的需求与代码生成覆盖率。

## 3. 本次范围（P0）
- 用户端工作空间详情页（流程闸门、按钮状态、文案提示、文件统计）。
- 用户端工作空间详情页 Token 额度可视化（总量/已用/剩余 + 进度条）。
- 代码生成接口防重复策略。
- 代码预览弹窗分组结构优化。
- 需求扩展与代码生成提示词的“双端约束”加强。
- Worker 兜底代码骨架扩展（SpringBoot/Node + Vue/React）。

## 4. 详细需求

### 4.1 AI 对话解锁策略
- 只有存在 `CODE` 类型文件后，才展示完整 AI 对话面板。
- 未生成代码时显示“待解锁”占位卡片，明确提示先完成代码生成。

### 4.2 生成代码按钮防重复
- 前端按钮状态：
  - 已生成代码后按钮显示“已生成”并禁用。
  - 正在运行任务时继续禁用（沿用全局操作锁）。
- 后端接口策略：
  - 若存在 `CODE` 文件，`/api/workspace/[id]/generate-code` 直接返回 `409`。
  - 若存在进行中的 `CODE_GEN` 任务，同样返回 `409`。

### 4.3 预览与下载一致性
- 预览弹窗代码文件分组：
  - 后端文件
  - 前端文件
  - SQL 脚本
  - 文档说明
  - 其他代码
- 右侧“项目文件”卡片新增说明：
  - 下载包包含当前工作空间的全部代码文件；
  - 预览仅用于查看，不影响下载范围。
- 右侧“项目源代码”新增结构计数：
  - 后端 / 前端 / SQL / 其他。

### 4.4 双端项目覆盖
- 需求扩展阶段默认按“双端系统”规划（用户端 + 管理端）。
- 代码生成提示词新增双端硬约束（命中双端场景时必须包含双端页面与接口）。
- Worker 兜底骨架升级：
  - SpringBoot 分支增加管理端与用户端接口示例、实体与基础配置文件。
  - Vue/React 分支增加用户端与管理端页面示例。
- 最低代码文件阈值从 `6` 提升到 `12`，减少“文件过少但误判成功”。

### 4.5 Token 用量进度展示
- 在工作空间右侧新增 `Token 用量` 卡片。
- 展示当前账号 Token 总量、已用量、剩余量。
- 使用进度条展示使用比例（`tokenUsed / tokenBudget`）。
- 数据来源：`GET /api/billing/tokens`（按用户维度聚合，不按单工作空间拆分）。

## 5. 验收标准
- 未生成代码时不显示可交互 AI 对话框。
- 已生成代码后再次点击“生成代码”不会创建新任务（接口返回 409）。
- 预览弹窗可看到后端/前端/SQL 分组。
- “项目源代码”区域可看到结构化数量统计。
- 新建双端语义明显的题目时，生成结果应包含用户端与管理端示例文件。

## 6. 非目标（本次不做）
- 不在本次实现“用户自定义重新全量再生成”流程。
- 不在本次实现“多模板多版本代码产物比较”。
- 不在本次实现“按模块增量生成代码”精细编排。

## 7. 风险与后续
- 由于模型输出存在波动，仍可能出现结构不完整，已通过“阈值 + 兜底骨架”降低风险。
- 后续可考虑在工作台增加“代码完整性评分”与“缺失模块提示”。
