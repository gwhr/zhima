# 软件类方向收敛 PRD（2026-03-29）

## 背景
- 当前版本产品能力以“信息系统/软件工程类毕设”最稳定。
- 用户在选题阶段可能输入算法研究类方向（如目标检测），会导致预期与实际可交付内容不一致。

## 目标
- 在“选题推荐”和“需求确认”链路中，统一收敛到信息系统/软件工程方向。
- 对算法研究型关键词自动转译为“结果管理与展示平台”类题目，避免输出训练/硬件方案。

## 变更范围
- `POST /api/ai/recommend-topics`
  - 增加方向强约束提示词。
  - 增加违规方向关键词过滤。
  - 当推荐结果不足时补充系统方向兜底题目。
- `POST /api/ai/expand-requirements`
  - 增加方向强约束提示词，明确只输出系统化需求。
- `POST /api/workspace/[id]/confirm-requirements`
  - 在难度评估与需求改写提示中加入方向约束上下文。
- 创建工作空间弹窗
  - 选题与需求确认步骤增加“仅支持软件类方向”的明确提示文案。

## 验收标准
- 用户输入算法类关键词时，推荐结果仍为信息系统/软件方向题目。
- 需求确认后展示的模块、角色、数据表围绕系统化交付，不出现训练脚本或硬件控制需求。
- 前端流程中可见方向限制提示，减少用户误解。

## 后续规划
- 新增“算法研究类毕设”独立方向（如目标检测）时，再开放方向切换与专用模板。
