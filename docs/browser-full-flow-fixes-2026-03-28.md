# 浏览器全链路回归与卡点修复（2026-03-28）

## 背景
在浏览器端全链路实测中，发现以下关键卡点：

1. `生成代码` 接口偶发 `429`，原因是历史 `PENDING/RUNNING` 任务未清理，占用并发额度。
2. 本地多开历史进程时，出现数据库连接数爆满，导致任务失败。
3. 创建项目弹窗存在 `DialogDescription` 缺失告警。
4. 注册页未显式要求同意《用户协议》《隐私政策》。

## 已完成修复

### 1) 并发风控僵尸任务自动清理
- 文件：`lib/risk-control.ts`
- 修复点：
  - 在并发校验前自动清理超时任务：
    - `PENDING` 超过 10 分钟 -> 标记 `FAILED`
    - `RUNNING` 超过 120 分钟 -> 标记 `FAILED`
  - 清理后再计算活跃任务数，避免用户被历史僵尸任务长期阻塞。
  - 修复并发超限提示文案为正常中文。

### 2) 注册页补齐协议同意
- 文件：`app/(auth)/register/page.tsx`
- 修复点：
  - 新增协议勾选区（用户协议/隐私政策链接）。
  - 未勾选时禁止邮箱注册、手机号注册、发送验证码。
  - 未勾选触发时给出明确提示。

### 3) 弹窗无障碍告警修复
- 文件：`components/create-workspace-dialog.tsx`
- 修复点：
  - 为 `DialogHeader` 增加 `DialogDescription`，消除 Radix Dialog 警告。

### 4) 新增可复用全链路回归脚本
- 文件：`scripts/e2e-full-flow.mjs`
- 覆盖流程：
  - 登录 -> 新建工作空间 -> 功能确认/难度评估 -> 生成代码 -> 预览通过 -> 生成论文 -> 下载 -> 管理端关键页面冒烟

## 说明
数据库连接数爆满属于本地多开进程环境问题，线上按单实例规范部署不会出现同类问题。为避免本地复现该问题，建议每次只保留一套 `web + worker` 进程运行。
