# 选题推荐技术栈前置与一致性 PRD（2026-03-31）

## 背景
- 现状流程中，用户先看“选题推荐”，再在下一步选择技术栈。
- 导致推荐题目展示的技术栈与用户后续选择不一致，产生理解偏差与返工。

## 目标
- 在“选题推荐”前置让用户选择技术栈。
- AI 推荐时同时带入“业务关键词 + 技术栈关键词”。
- 推荐结果中的 `techStack` 与用户选择保持一致，避免错配。

## 方案
1. 前端（创建工作空间弹窗）
- 在 `keyword` 步骤增加：
  - 专业分类选择
  - 后端/数据库/前端技术栈选择
- 搜索推荐请求新增 `techStack` 参数。
- 在推荐结果区域展示当前推荐技术栈，明确“按该技术栈推荐”。

2. 后端（`/api/ai/recommend-topics`）
- 接收并校验 `techStack`（非法值回退默认值）。
- 在 Prompt 中加入技术栈硬约束。
- 将返回题目的 `techStack` 统一为用户选中的组合。
- 兜底推荐（fallback）同样使用用户所选技术栈。

## 验收标准
- 用户在“选题推荐”页改动技术栈后，再点搜索，推荐结果需体现同一技术栈。
- “再来一批”仍保持同一技术栈约束，不出现跨栈题目。
- 进入下一步“选择技术栈”时，默认值与前一步保持一致。

## 非目标
- 本次不新增技术栈种类，仅沿用当前已支持列表。
- 本次不改变后续代码生成/论文生成模型策略。
