# 首页卖点增强与一对一指导二维码占位（PRD）

日期：2026-04-02  
状态：已落地（本地），待线上回归确认

## 背景

当前首页信息量偏少，毕设定位和价值表达不够突出；同时一对一指导入口在未配置二维码时缺乏可展示内容，影响转化与信任感。

## 目标

1. 首页强化“毕设助手 / 毕设辅导”定位，突出平台是辅助工具而非代做。
2. 补充卖点亮点内容，帮助用户快速理解产品价值和使用路径。
3. 一对一指导先提供可展示二维码占位图，避免空状态。
4. 保持后续可配置能力：管理员可在后台替换为正式客服二维码。

## 改动范围

### 1) 首页内容升级

文件：`app/(marketing)/page.tsx`

- 新增“定位亮点”区块（辅导定位、全链路协同、过程可控）。
- 新增“使用流程”区块（选题 -> 代码 -> 预览讨论 -> 论文整理）。
- 新增“交付能力”区块（源码预览下载、论文图表、状态反馈、一对一辅导入口）。
- 新增“一对一辅导”展示卡（含二维码占位图）。
- Hero 区文案强化“你主导，AI 辅助”与合规表达。

### 2) 一对一二维码占位图

文件：`public/support-qr-placeholder.svg`

- 新增默认占位二维码图片，用于未配置正式客服二维码时展示。

### 3) 平台默认配置调整

文件：`lib/system-config.ts`

- `supportContactEnabled` 默认值从 `false` 调整为 `true`。
- `supportContactQrUrl` 默认值从空字符串调整为 `/support-qr-placeholder.svg`。

### 4) 环境变量示例同步

文件：
- `.env.example`
- `.env.production.example`

新增默认建议值：
- `SUPPORT_CONTACT_ENABLED=true`
- `SUPPORT_CONTACT_QR_URL=/support-qr-placeholder.svg`

## 验收标准

1. 首页能清晰传达“毕设辅导/助手”定位，不出现“平台替用户完成学业”的表述。
2. 首页存在可识别的卖点亮点区块，信息密度明显高于旧版。
3. 未配置客服二维码时，页面仍显示二维码占位图，不出现空白。
4. 管理后台后续配置真实二维码后，前台可无代码替换展示。

## 风险与回滚

- 风险：文案密度提升后，移动端信息层级可能偏重。
- 处理：保留原有卡片结构与响应式栅格，必要时再做移动端压缩文案版本。
- 回滚：可回退 `page.tsx` 与 `support-qr-placeholder.svg` 两处即可恢复原状。
