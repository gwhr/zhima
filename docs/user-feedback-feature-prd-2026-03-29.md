# 用户反馈（图文）功能 PRD（2026-03-29）

状态：已落地（用户端提交/列表、管理端分页检索/处理、图片鉴权均已实现）

## 目标

为普通用户提供“文字 + 图片”反馈通道，并让管理员在后台统一查看、检索和处理反馈。

## 功能范围

### 用户侧

- 新增页面：`/dashboard/feedback`
- 支持填写反馈内容（必填）
- 支持填写联系方式（选填）
- 支持上传图片（最多 3 张，单张 <= 2MB）
- 支持查看“我的反馈记录”
- 反馈状态显示：`待处理` / `已处理`
- 当管理员填写备注后，用户可看到“管理员回复”

### 管理员侧

- 新增页面：`/admin/feedback`
- 支持分页查看反馈列表
- 支持关键字检索（内容/联系方式/用户名/邮箱/手机号）
- 支持状态筛选（全部/待处理/已处理）
- 支持标记处理状态（待处理 <-> 已处理）
- 支持填写管理员备注
- 支持查看反馈附图

## 数据结构

新增表：`UserFeedback`

- `id`
- `userId`
- `content`
- `contact`
- `pagePath`
- `imageKeys`（字符串数组）
- `status`（`OPEN` / `RESOLVED`）
- `adminNote`
- `resolvedAt`
- `resolvedById`
- `createdAt` / `updatedAt`

## API 设计

- `POST /api/feedback/upload`：上传反馈图片
- `GET /api/feedback`：获取当前用户反馈列表
- `POST /api/feedback`：提交反馈
- `GET /api/admin/feedback`：管理员分页查询反馈
- `PATCH /api/admin/feedback`：管理员更新状态和备注
- `GET /api/feedback/image/[...key]`：受控读取反馈图片（用户本人或管理员）

## 安全与风控

- 上传文件类型限制：`jpg/png/webp/gif`
- 上传体积限制：单张 2MB
- 图片访问鉴权：仅反馈创建者和管理员可查看
- 管理员状态更新写入审计日志

## 验收标准

1. 用户可提交至少 1 条图文反馈，并在“我的反馈记录”中看到。
2. 管理员能在后台检索到该反馈并查看图片。
3. 管理员改为“已处理”后，用户端状态同步为“已处理”。
4. 管理员备注可在用户端展示。
5. 非本人不可访问他人反馈图片。
