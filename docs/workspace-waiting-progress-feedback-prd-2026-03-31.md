# 工作空间步骤等待态过程反馈 PRD（2026-03-31）

## 背景
- 用户在“生成代码/生成论文”阶段等待时间较长时，容易误判页面卡死。
- 现有仅展示进度条与简短文案，过程感不足。

## 目标
- 在步骤卡中提供可感知的“实时过程反馈”，提升等待期可理解性与信任感。

## 改动内容
1. 进行中步骤增加动态阶段提示
- 为 `CODE_GEN / THESIS_GEN / PREVIEW` 提供阶段提示序列。
- 当后端未持续返回阶段时，前端按进度与时间轮播显示阶段提示，避免静态停滞感。

2. 展示任务运行时长
- 在进行中状态显示“已运行 xxs / xmxs”，提示任务仍在执行。

3. 展示待解锁原因
- 对未开始的步骤增加“为什么现在不能点”的解释文案，例如：
  - 等待步骤1完成后解锁步骤2
  - 先完成预览确认后再生成论文

## 验收标准
- 代码生成执行时，步骤卡至少显示：阶段提示、进度、运行时长。
- 步骤2/3未解锁时，用户可看到明确原因，不再“无反馈等待”。
- 不影响现有任务状态轮询与按钮权限逻辑。
